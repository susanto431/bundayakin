360 Apps — Design System Audit

Analisis sistematis design system 360 Apps yang dibangun di atas Vue 3 + Inertia.js + Tailwind CSS dengan token shadcn-style. Mengidentifikasi konflik token, duplikasi komponen, dan poin migrasi sebagai landasan konsolidasi sistem.

Codebase · 360-apps
Stack · Vue 3 · Inertia · Tailwind v3
UI Lib · Radix Vue · Headless UI
Tanggal · 2026-05-19

Ringkasan Eksekutif

Design system dibangun di atas Tailwind CSS dengan token berbasis CSS custom properties (shadcn pattern), namun implementasinya tidak konsisten. Ditemukan 3 nilai hex berbeda untuk warna primary, 1 bug kritis pada token --primary-foreground, dan 4 implementasi Select dengan library & style berbeda. Total: 6 inkonsistensi kritis dan minimal 9 poin duplikasi yang perlu dikonsolidasi.

Rekomendasi

Action item terprioritisasi untuk konsolidasi design system.

Prioritas Tinggi

1. Fix bug --primary-foreground — app.css:22: nilainya saat ini sama dengan --primary (169, 99%, 38%). Seharusnya warna terang agar teks di atas tombol primary terbaca. Ganti ke 0 0% 100% untuk kontras optimal. Ini bug fungsional yang mempengaruhi aksesibilitas.

2. Standardisasi satu nilai warna primary. Saat ini ada 4 variasi (hsl(169,99%,38%), #01c3a0, #01c19e, #39B597). Putuskan satu nilai kanonik, masukkan ke token --primary, lalu ganti semua hardcode. Lokasi yang perlu diubah:

  • app.css:99
  • app.css:111
  • app.ts:110
  • navigation-menu/index.ts:13
  • settings/Layout.vue:215

3. Konsolidasi komponen Select. 4 implementasi berbeda (Select.vue, SelectNew.vue, SelectV2.vue, SelectV3.vue) menyebabkan inkonsistensi visual (biru vs hijau) dan bundle size membengkak. Paling kritis: SelectNew.vue memiliki warna hover/selected biru #1d4ed8 yang bertentangan dengan brand.

4. Fix missing semicolon di CSS token. app.css:16: --black-text: 358 0% 10% perlu tambah ; di akhir baris. Token ini bisa tidak dibaca browser di beberapa konfigurasi.

5. Tambahkan token warna yang hilang. Buat token untuk warna yang berulang tapi tidak bertokenisasi:

--warning: hsl(43, 100%, 47%);        /* #ffb800 */
--primary-subtle: hsl(169, 71%, 93%); /* #e6f9f6 */
--disabled-bg: hsl(0, 0%, 95%);       /* #f3f3f3 */
--table-accent: hsl(190, 60%, 40%);   /* #278796 */

Ini akan menggantikan setidaknya 8 nilai hardcode yang tersebar.

Prioritas Sedang

1. Perbaiki hierarki Heading. HeadingSmall.vue menggunakan text-2xl sedangkan Heading.vue menggunakan text-xl. Secara semantik H3 seharusnya lebih kecil dari H2. Ganti HeadingSmall.vue:11 ke text-xl dan Heading.vue:16 ke text-2xl, atau sesuaikan nama komponen.

2. Hapus komponen legacy. components/ui/Sidebar.vue (yang lama) sudah ada penggantinya di components/ui/sidebar/ kit. Hapus file lama dan pastikan tidak ada import yang tersisa.

3. Migrasi Modal ke Dialog kit. components/Modal.vue memakai Headless UI sementara components/ui/dialog/ memakai Radix Vue. Pilih satu sistem. Modal.vue juga menggunakan bg-white hardcode di Modal.vue:37 — ganti ke bg-background.

4. Ganti inline style ke Tailwind. Tiga file (Schedule/Form.vue, AppLogo.vue) menggunakan style="" yang bisa diganti Tailwind class.

5. Buat skala z-index eksplisit. Tambahkan komentar di tailwind.config.js atau app.css yang mendokumentasikan z-index scale: overlay: 50, modal: 100, dropdown: 200, tooltip: 9999. Saat ini nilai 9999 dan 101 tersebar tanpa pola.

Catatan untuk Tim Dev

  • bg-[#fafafa] dipakai di 5 tempat (AppHeader, 3 layouts, settings). Nilai ini identik dengan --sidebar-background: 0 0% 98%. Ganti ke class Tailwind bg-sidebar untuk konsistensi.
  • disabled:bg-[#f3f3f3] di Input.vue, MoneyInput.vue, dan PhoneInput.vue perlu disamakan — saat ini Input pakai #f3f3f3 sementara SelectNew pakai #f3f4f6. Pertimbangkan buat token --input-disabled.
  • Chart.js di Overview.vue dan Detail.vue menggunakan warna rgba hardcode yang tidak berhubungan dengan token chart yang ada di tailwind.config.js. Pertimbangkan ekstrak ke konstanta di file terpisah.
